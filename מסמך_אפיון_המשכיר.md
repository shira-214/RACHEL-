# מסמך אפיון — מערכת "המשכיר"

**פרויקט גמר | ניהול והזמנת דירות לשכירות קצרה**  
**גרסה:** 1.0 | **תאריך:** יולי 2026

---

## 1. תקציר מנהלים

"המשכיר" היא מערכת Desktop לניהול והזמנת דירות לשכירות קצרה (סגנון Airbnb מקומי).  
המערכת משרתת **שלושה סוגי משתמשים**: שוכר, משכיר (בעל דירה) ומנהל מערכת.

הארכיטקטורה מבוססת **Client–Server קלאסי**:
- **לקוח:** WPF (C#, .NET Framework 4.7.2)
- **שרת:** WCF (Windows Communication Foundation)
- **לוגיקה עסקית:** BL
- **גישה לנתונים:** DAL + Entity Framework 6
- **מסד נתונים:** SQL Server LocalDB (`.mdf`)

## 4. מודל נתונים (10 ישויות)

| # | ישות | תיאור | מפתח |
|---|------|--------|------|
| 1 | **Areas** | אזורים גיאוגרפיים (צפון, מרכז, דרום) | IdArea |
| 2 | **Cities** | ערים | IdCity |
| 3 | **StreetsNames** | שמות רחובות | IdStreet |
| 4 | **AraesCitiesStreet** | קישור N:M בין אזור, עיר ורחוב | מפתח מורכב |
| 5 | **Apartments** | דירות לשכירות | IdApartment |
| 6 | **Extras** | תוספות (WiFi, חניה, מיזוג…) | IdExtra |
| 7 | **ExtrasApartements** | קישור תוספת↔דירה | IdExtra + IdApartment |
| 8 | **Images** | תמונות לדירה (שם קובץ + סטטוס) | IdApartment + NumImage |
| 9 | **Hirers** | שוכרים | C_IDHirer (ת.ז.) |
| 10 | **Rentings** | הזמנות / השכרות | IdRenting |

### קשרים עיקריים

- **Apartment** ← City, Street, Images, ExtrasApartements, Rentings
- **Renting** ← Hirer, Apartment
- **AraesCitiesStreet** ← Area, City, Street (לסינון גיאוגרפי בחיפוש)

### שדות חשובים — Apartment

| שדה | משמעות |
|-----|--------|
| NameOwner, PhoneOwner | זיהוי משכיר (כניסה) |
| MinimumPrice | מחיר בסיס ללילה |
| ExtraForBed | תוספת מחיר לכל מיטה |
| NumberBeds | קיבולת מקסימלית |
| Status | האם הדירה פעילה ומוצגת בחיפוש |
| note | הערות חופשיות |

## 6. זרימות משתמש (Use Cases)

### UC-01: חיפוש והזמנת דירה (שוכר)

1. שוכר מתחבר בת.ז.
2. נכנס למסך חיפוש (`PSearch`).
3. מסנן לפי: אזור, עיר, רחוב, מחיר מקסימלי, מיטות מינימום, תאריך, תוספת.
4. רואה כרטיסי דירות (`USCApartement`) — תמונה, כתובת, מחיר, כפתור "הזמן עכשיו".
5. מזין: מספר מיטות, תאריך, פרטי אשראי (ולידציה בלבד).
6. המערכת בודקת: דירה פנויה, מיטות בקיבולת, אשראי תקין.
7. נוצרת רשומת `Renting`; השוכר מועבר חזרה לחיפוש.

### UC-02: ניהול דירה (משכיר)

1. משכיר מתחבר בשם + טלפון.
2. רואה לוח בקרה עם דירותיו והזמנות.
3. יכול: להוסיף דירה, לערוך דירה, להוסיף תמונה.
4. תמונה: בחירה מהמחשב → תצוגה מקדימה → "שמור תמונה" → קובץ ב-`Pictures/` + רשומה ב-DB.

### UC-03: ניהול מערכת (מנהל)

1. מנהל מתחבר admin/1234.
2. נכנס לאזור ניהול עם תפריט 10 ישויות.
3. לכל ישות: רשימה (`All*`), הוספה (`AddManager*`), עדכון, מחיקה.
4. מחיקה עם תלויות (FK) — הודעת שגיאה בעברית.

## 7. כללי עסק (Business Rules)

| # | כלל | נוסחה / התנהגות |
|---|-----|-----------------|
| BR-01 | מחיר הזמנה | `SumPayment = MinimumPrice + (ExtraForBed × SumBeds)` |
| BR-02 | זמינות | דירה תפוסה אם קיימת הזמנה באותו **יום** (`Date`) |
| BR-03 | הצגה בחיפוש | רק דירות עם `Status = true` |
| BR-04 | תמונה בכרטיס | תמונה **פעילה** (`Stataus = true`) עם bytes תקינים |
| BR-05 | אשראי | ולידציה: 16 ספרות, CVV 3 ספרות, תוקף MM/YY — **ללא חיוב אמיתי** |
| BR-06 | תמונות | קובץ físico ב-`Pictures/`; DB שומר **שם קובץ** בלבד |
| BR-07 | מחיקה | סדר מומלץ: הזמנות → תמונות → תוספות-דירה → דירה |

## 13. מצב נוכחי (לאחר תיקונים)

### מה עובד

- שלושת תפקידי המשתמש עם הפרדת הרשאות
- CRUD מלא למנהל על 10 ישויות
- חיפוש עם סינון (אזור, עיר, רחוב, מחיר, מיטות, תאריך, תוספת)
- הזמנה עם ולידציית אשראי
- העלאת תמונות (בחירה + שמירה ל-DB ול-`Pictures/`)
- תיקוני encoding עברית, bindings, StackOverflow, נתיב תמונות

### מגבלות ידועות

| מגבלה | הסבר |
|--------|------|
| Session בזיכרון | אין persistence — סגירת אפlikation מאפסת |
| מנהל ב-config | סיסמת admin לא ב-DB, לא ניתנת לשינוי מה-UI |
| תשלום מדומה | אין חיבור לסליקה אמיתית |
| LocalDB | DB לא משותף בין מחשבים (ללא Git) |
| WCF legacy | טכנולוגיה ישנה, לא cloud-native |
| שמות שדות | `Stataus`, `FhoneNumberHirer` — שגיאות כתיב ב-DB |
| ComboBox דירה (מנהל) | `DisplayMemberPath=NumberHouse` — לא תמיד ייחודי |
